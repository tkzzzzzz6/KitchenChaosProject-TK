# KitchenChaosProject（小黄鸭厨房闯关）

[中文](README.md) | [English](README.en.md)

## 游戏介绍

故事从一张“欠费通知单”开始：咔哒原本只是厨房里的小帮手，但为了保住这间快餐小店，它决定独自顶上最忙的班次。客人们不等人，食材也不能浪费——每一份汉堡都必须按时完成：先准备面包与配菜，再把肉饼煎到恰到好处，最后快速组装并递送。你需要在有限空间里规划路线，减少无效走动，把烹饪与组装衔接起来，才能稳定应对高峰时段。

这是一款轻松有趣的单人厨房闯关游戏。你将操控一只圆滚滚的小黄鸭厨师(咔哒)在一间快节奏的小厨房里迎接不断到来的客人订单：取食材、切配、烹饪、组装汉堡，并在规定时间内把成品送到客人手中。订单会持续刷新，节奏逐步加快，考验你的操作效率与步骤安排。

游戏的核心在于“顺序与规划”：先做哪一单、先煎还是先切、什么时候递送最合适，都会直接影响完成数量与最终评价。随着你逐渐熟悉厨房节奏，咔哒也会从手忙脚乱的新手，成长为能在高压下依旧稳稳出餐的“鸭鸭主厨”，用一份份热乎乎的汉堡赢回口碑与掌声。

## 项目结构（主要目录说明）

> 仅说明主要文件夹的位置与用途，不逐个解释文件。

- `Assets/`
  - Unity 工程的主要资源目录：包含场景、脚本、预制体、美术/音频等游戏内容。
- `Assets/Scripts/`
  - 项目脚本目录：主要 C# 代码（玩法、交互、UI、系统逻辑等）。
- `Packages/`
  - Unity Package Manager 配置与依赖（项目使用的包）。
- `ProjectSettings/`
  - Unity 项目设置（输入、渲染、质量、标签层级等配置）。
- `UserSettings/`

  - 本机/用户侧的编辑器设置（通常不影响项目逻辑，且多与个人环境相关）。

- `Library/`（生成目录）
  - Unity 自动生成的缓存与导入产物，通常不需要手动修改。
- `Logs/`
  - Unity/编辑器相关日志输出。
- `build/`

  - 本项目的构建输出目录（可执行包体/平台产物等）。

- `openspec/`
  - 项目规范与协作流程相关内容（如改动提案、约定等）。
